Update your template

We are continually improving the California State Web Template. View release features by date for items that have been added or updated.

If you are new here, you may find our getting started page helpful.

Find changelogs for V6 and earlier on the GitHub production release page

  1. Save a backup

    Before updating to the latest State Web Template version, save a backup of your current site. Use the backup to restore your current site if the update fails.

  2. Download the latest State Web Template release

    Use our GitHub repository to download the latest release. If you are using the State Web Template CDN, update your CDN links to the latest release.

  3. Compare release changes

    Use GitHub to compare releases and view code changes at a glance. This works best if you are already using a published GitHub version of the state template.

    You can also locally compare code files to the latest release.
  4. Inspect CSS and JavaScript changes

    Most changes are made in the CSS and JavaScript source files in the /src/scss/ and /src/js/ folders. Inspect those files thoroughly. Update your source CSS and JavaScript files if necessary.

  5. Check for structural changes

    Look for structural code changes to your components in one of these folders:

    ● State Web Template /src/_includes/

    ● State Web Template /src/_includes/modules/

    Update your code if necessary.
  6. Copy site fonts

    Copy the fonts files from /src/fonts/ folder.
  7. Build and test your site

    Build and run your site locally. Make sure there are no NPM (Node Package Manager) or other errors when using build commands. Perform manual and automatic accessibility testing of all updated pages.
